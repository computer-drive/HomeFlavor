# 数据库设计

## `orders` 表设计
1. `id`
    - 主键，自动递增
    - 系统内部使用，唯一标识每个订单。
2. `order_num`
    - 整数
    - 唯一标识每个订单。
    - 用于用户查询和跟踪订单的状态。每日订单号从1开始递增。
3. `table_num`
    - 整数
    - 订单所属的桌号。
    - 用于记录订单所属的桌号，方便用户查询和管理。
4. `status`
    - 文本(pending, cooking, done, canceled, paid)
    - 订单状态。
    - 用于记录订单的当前状态，有待处理、制作中、已完成，已取消，已结账。
    - 默认值为`pending`。
5. `items_json`
    - 文本
    - 订单中的菜单项，JSON格式存储。
    - 用于记录订单中的菜单项，方便查询和管理。
    - 格式
    ```json
    [
    {
        "id": 1, // 菜品ID，唯一标识每个菜品
        "name": "菜品名称",
        "price": 1200, // 菜品单价，单位：分
        "quantity": 1, // 菜品数量，默认值为1
        "added": {     // 是否是后来加的，此项可选，若非后加，则可省略
            "time": "2023-12-12 12:00:00", // 加入时间
            "by": 0 // 店员ID
        }
        
    }
]
    ```
6. `total_price`
    - 整数
    - 订单总金额，单位：分
    - 用于记录订单的总金额，方便用户查询和管理。

## `menu`表设计
1. `id`
    - 主键，自动递增。
    - 系统内部使用，唯一标识每个菜品。
2. `name`
    - 文本
    - 菜品名称。

3. `price`
    - 整数
    - 菜品单价，单位：分。

4. `category`
    - 文本
    - 菜品分类。
5. `description`
    - 文本
    - 菜品描述。
6. `image_url`
    - 文本
    - 菜品图片URL。
7. `is_available`
    - 整数
    - 是否可用，默认值为1（可用）。
8. `options_json`
    - 文本
    - 菜品可选配置，JSON格式存储。
    - 格式
    ```json
        [
        {
            "name": "选项名称",
            "choice": [
                { // 若该选需要额外的费用，则使用对象
                    "name": "选项名称", 
                    "price": 100
                },
                "选项名称"  // 若该选不需要额外的费用，则可直接写选项名称
            ]
        }
    ]
    ```
